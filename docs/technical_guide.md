## 1. プロジェクト構成とモジュール設計

本アプリケーションは、Streamlit を用いた対話型 UI と、spaCy / GiNZA をベースとした自然言語処理 (NLP) エンジン、統計分析コア、可視化エンジンにモジュール化されています。

### 1.1. ファイル構成
```text
src/
├── text_mining_app.py        # Streamlit UI エントリーポイント（画面描画、操作イベント処理）
└── core/
    ├── __init__.py           # パッケージ初期化
    ├── config.py             # 設定値管理・状態管理・パス解決 (`resource_path`)
    ├── nlp_engine.py         # spaCy/GiNZA を用いた形態素解析・感情極性付与
    ├── stats.py              # 分析アルゴリズム（TF-IDF、感情スコア、共起、N-gram、対応分析）
```

### 1.2. エンジニアリング上の主要機能
* **モデルロードのキャッシュ化**
  自然言語処理モデル（`ja_ginza`）の読み込みは実行コストが高いため、[nlp_engine.py](file://src/core/nlp_engine.py) にて `@st.cache_resource` を使用して初回のみロードするようにキャッシュ化されています。

  アプリをスタンドアロンの実行ファイルとしてビルドした際、一時ディレクトリ（`_MEIPASS`）に展開されるリソース（日本語フォント `NotoSansJP-Regular.ttf` や感情極性辞書 `sentiment_dict.csv`）の参照を正しく通すため、[config.py](file://src/core/config.py) 内に `resource_path` 関数が定義されています。

## 2. テキストマイニングの基本プロセス

生の日本語文章から分析結果を可視化するまでのプロセスは、以下のステップで実行されます。

```text
  [入力テキスト]
       │
       ▼
1. 形態素解析 (nlp_engine.py)
   ├─ spaCy/GiNZA による文境界の検出、単語分割
   ├─ 品詞(POS)判定と基本形(lemma)の抽出
   └─ 感情極性辞書とのマッピング
       │
       ▼
2. フィルタリングと前処理 (stats.py)
   ├─ 類義語（シノニム）置換の適用
   ├─ 文字標準化（アルファベット小文字化、全角半角の標準化）
   ├─ ユーザー指定品詞のフィルタリング
   └─ ストップワード（除外ワード）の除去
       │
       ▼
3. 統計計算・分析ロジック (stats.py)
   ├─ 出現回数（頻出度）の集計
   ├─ TF-IDF による単語重要度スコアリング
   ├─ N-gram 連語の抽出 (Bigram / Trigram)
   ├─ 単語間の共起度（Jaccard係数）の計算
   ├─ 文単位の感情スコアリング（否定表現の反転考慮）
   └─ 属性と単語の対応分析 (SVD: 特異値分解)
       │
       ▼
4. インタラクティブな可視化 (visualizer.py)
   └─ WordCloud、Plotly（棒グラフ、散布図、共起ネットワーク）での描画
```

## 3. 分析アルゴリズムの数理と仕様

### 3.1. TF-IDF (単語重要度スコア)

TF-IDF (Term Frequency-Inverse Document Frequency) は、ある文書群（コーパス）において、特定の単語が「どれだけその文書を特徴づけているか」を評価する指標です。

#### 数理モデル
1. **TF (Term Frequency) : 単語の出現頻度**
   ある単語 $t$ が文書 $d$ 内で出現した割合です。文書の長さに依存しないよう、総単語数で正規化します。
   $$tf(t, d) = \frac{n_{t,d}}{\sum_{k} n_{k,d}}$$
   * $n_{t,d}$ : 文書 $d$ における単語 $t$ の出現回数
   * $\sum_{k} n_{k,d}$ : 文書 $d$ における総単語数

2. **IDF (Inverse Document Frequency) : 逆文書頻度**
   その単語 $t$ が全文書集合 $D$ の中でどれだけ「珍しいか」を示します。
   $$idf(t) = \log\left(\frac{N}{df(t)}\right)$$
   * $N$ : 全文書数 ($|D|$)
   * $df(t)$ : 単語 $t$ が出現する文書の数

   > [!NOTE]
   > **平滑化 (Smoothing)**
   > 本アプリの実装（`scikit-learn` の `TfidfVectorizer`）では、ゼロ除算（$df(t)=0$）を防ぐため、分母分子に $1$ を加算し、対数全体に $1$ を加算する以下の平滑化式を使用しています。
   > $$idf(t) = \log\left(\frac{N + 1}{df(t) + 1}\right) + 1$$

3. **TF-IDF スコアの算出**
   $$\text{tfidf}(t, d, D) = tf(t, d) \times idf(t)$$

#### 具体的な計算プロセス（3文書の例）
*(平滑化なし、自然対数 $\ln$ を用いた手計算)*
* **文書A**: 「猫 の ご飯 は、美味しい ご飯 です。」 (総単語数: 8)
* **文書B**: 「犬 の ご飯 も、美味しい です。」 (総単語数: 6)
* **文書C**: 「猫 の 散歩 は 大変 です。」 (総単語数: 7)

| 単語 $t$ (対象文書) | TF (in 対象文書) | IDF (in 全文書) | TF-IDF スコア |
| :--- | :--- | :--- | :--- |
| **猫** (文書A) | $1 / 8 = 0.125$ | $\ln(3/2) \approx 0.405$ | $0.125 \times 0.405 \approx \mathbf{0.051}$ |
| **美味しい** (文書A) | $1 / 8 = 0.125$ | $\ln(3/2) \approx 0.405$ | $0.125 \times 0.405 \approx \mathbf{0.051}$ |
| **ご飯** (文書A) | $2 / 8 = 0.250$ | $\ln(3/2) \approx 0.405$ | $0.250 \times 0.405 \approx \mathbf{0.101}$ |
| **散歩** (文書C) | $1 / 7 \approx 0.143$ | $\ln(3/1) \approx 1.098$ | $0.143 \times 1.098 \approx \mathbf{0.157}$ |

* **考察**: 「ご飯」は出現回数が多い（TFが高い）ですが、他の文書にも出現するため、文書Cにのみ出現する「散歩」の方が、IDFの効果によって最終的なTF-IDF重要度スコアが高くなります。

### 3.2. 感情分析 (否定語を考慮した極性判定)

単純な辞書マッチングだけでなく、日本語の係り受け（修飾関係）を利用して、**否定表現が伴う場合に感情極性を反転させる** 高度な処理を実装しています。

#### 感情分析の処理手順
1. 東北大学 乾・関根研究室の「日本語極性評価辞書」を用いて、形態素解析時に単語ごとにポジティブ (`p`) / ネガティブ (`n`) を付与。
2. spaCyの依存関係木（係り受け構造）から、対象単語の子ノード（修飾語など）を探索。
3. 修飾関係に否定の助動詞・動詞（`ない`, `ぬ`, `ず`, `かねる`, `せん`, `無い`, `なし` など）が含まれる場合、`is_negated` フラグを `True` に設定。
4. `is_negated` が `True` の場合、感情極性を反転（`p` → `n` / `n` → `p`）。
   * 例: **「美味しくない」**
     * 「美味しい」 (ポジティブ `p`) ＋ 「ない」 (否定) ＝ **ネガティブ (`n`)** として集計される。
5. **文単位の感情スコア計算**:
   文内のポジティブ単語数 ($pos$) とネガティブ単語数 ($neg$) から、文の感情スコアを決定します。
   $$\text{Score} = \frac{pos - neg}{pos + neg}$$
   * $\text{Score} > 0.05$ : ポジティブ文
   * $\text{Score} < -0.05$ : ネガティブ文
   * それ以外 : ニュートラル文

### 3.3. 共起ネットワーク (Jaccard係数)

文章内で「どの単語とどの単語が同じ文脈（一文中）で一緒に使われやすいか」を可視化します。

#### 数理モデル
単語 $A$ と単語 $B$ の関連性を評価するため、出現件数の積ではなく **Jaccard係数** を算出します。これにより、頻出すぎる一般単語の影響を抑え、結びつきの強さを正しく評価します。
$$\text{Jaccard}(A, B) = \frac{df(A \cap B)}{df(A) + df(B) - df(A \cap B)}$$
* $df(A)$ : 単語 $A$ が出現する文の数
* $df(B)$ : 単語 $B$ が出現する文の数
* $df(A \cap B)$ : 単語 $A$ と単語 $B$ が**同じ文**に共に出現する文の数

> [!TIP]
> **実装上の高速化工夫**
> 共起件数の算出には、`scikit-learn` の `CountVectorizer` を用いて、各文の単語出現状況を二値の疎行列 $X$ (Sparse Matrix) に変換した上で、行列積 $C = X^T \times X$ を一括演算することで、大規模テキストでも一瞬で共起回数を算出する最適化が行われています。

### 3.4. 対応分析 (SVDによる属性セグメント分析)

アンケートの「年代」「性別」や、レビューの「星の数」などの**属性（セグメント）情報と、出現単語の間の関係性を2次元マップに投影**し、視覚的に相関関係を把握します。

#### 計算アルゴリズム
1. **クロス集計表の作成**: 単語を行、属性を列とし、それぞれの出現頻度をカウントした行列 $X$ を作成。
2. **確率行列への変換**: 総和 $N$ で割り、同時確率行列 $P = X / N$ を算出。
3. **標準化残差行列 $S$ の計算**:
   $$S = D_r^{-1/2} (P - r c^T) D_c^{-1/2}$$
   * $r$ : 行の周辺確率ベクトル（行和）
   * $c$ : 列の周辺確率ベクトル（列和）
   * $D_r, D_c$ : それぞれ $r, c$ を対角成分とする対角行列
4. **特異値分解 (SVD: Singular Value Decomposition)**:
   $$S = U \Sigma V^T$$
   特異値分解により、標準化残差行列を分解し、上位2次元（最大特異値に対応する2つの固有ベクトル）を抽出します。
5. **座標 of 単語/属性 の算出**:
   * 行（単語）の座標: $R = D_r^{-1/2} U_2 \Sigma_2$
   * 列（属性）の座標: $C = D_c^{-1/2} V_2 \Sigma_2$
   これらを同一の2次元散布図にプロットすることで、「どのセグメントでどのようなキーワードが特徴的に使われているか」を同一の距離空間で俯瞰できます。
