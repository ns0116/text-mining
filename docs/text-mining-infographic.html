テキストマイニングって何？

アンケートやレビューなど、人間の書いた大量の文章データの中から「みんなが何て言ってる？」「どんな隠れた傾向がある？」をコンピュータの力で探し出す技術です。本アプリは、単なる頻度集計を超えた高度な統計・感情分析モデルを搭載しています。

このアプリでできる６つの高度な分析

頻度

1. 頻出度分析

どの言葉が何回登場したかを数えて話題の中心を把握します。名詞や動詞など、知りたい品詞だけに絞り込めます。

頻出語を Plotly で見やすくグラフ化
統計

2. TF-IDF 重要度分析

一般的な単語（「思う」「する」など）の重みを下げ、その文書を強く特徴づけるユニークなキーワードを抽出します。

「登場頻度」×「レア度」の重要度スコア
感情

3. 感情分析

極性辞書に基づき、文章がポジティブかネガティブかを判定。「美味しくない」などの否定語を検知し極性を自動反転します。

係り受け解析による否定表現の反転対応
関係

4. 共起ネットワーク

同じ文の中で「一緒に使われやすい単語」を線で結んだネットワーク図を描画。単語間の意味のつながりを視覚化します。

Jaccard係数に基づく意味の連関度
連続

5. N-gram 連語分析

2つの連続する言葉（Bigram）や3つの連続（Trigram）を抽出。「コール・センター」など、単語単位では見えない複合的な概念を捉えます。

頻出するフレーズや定型句の抽出
属性

6. 対応分析 (属性クロス)

アンケートの「性別」「年代」などの属性情報と単語の相関を特異値分解（SVD）で分析し、同一の2次元マップ上にプロットします。

どのセグメントが何に関心があるかを一目で把握

アルゴリズム解説①：感情極性の反転

日本語の係り受け文脈を考慮した高精度な感情スコアリング

一般的な感情分析
「ご飯は 美味しい が、
サービスは 良く ない」
美味しい (+) 良い (+)
「良い」の直後の「ない」を無視するため、ポジティブ過多に誤判定
当アプリの否定語反転 (v2.0)
「サービスは 良く ない」
良い (原本: +) ＋ ない (否定) ＝ ネガティブ (-) に反転
文脈木を探索し、否定語の係り受けを発見して正確に反転処理を行います。

アルゴリズム解説②：TF-IDF スコアの計算

単語の「出現頻度」と「希少性（レア度）」の数理モデルと計算例

TF

出現頻度 (Term Frequency)

その文書の中で、その単語がどれくらい出現したかの割合。文書の長さで正規化します。

tf(t, d) =
nt,d ∑k nk,d
IDF

逆文書頻度 (Inverse DF)

全文書におけるその単語の「珍しさ」。多くの文書に出現する一般的すぎる言葉の重みを下げます。

idf(t) = log(
N + 1 df(t) + 1
) + 1

※ 上記はゼロ除算とスコアの0化を防ぐ平滑化を施した標準的な計算式です。

TF-IDF

TF-IDF スコア

出現頻度(TF)と単語のレア度(IDF)を掛け合わせることで、文書の「特徴キーワード」を浮き彫りにします。

tfidf = tf(t, d) × idf(t)

具体的な計算プロセス（3つの文書を分析する場合）

文書A: 「猫 の ご飯 は、美味しい ご飯 です。」 (総単語数: 8)

文書B: 「犬 の ご飯 も、美味しい です。」 (総単語数: 6)

文書C: 「猫 の 散歩 は 大変 です。」 (総単語数: 7)

単語 (対象文書) TFの計算 IDFの計算 (平滑化なし・自然対数ln) 最終 TF-IDF スコア
猫 (文書A) 1 / 8 = 0.125 df(猫)=2 → ln(3/2) ≈ 0.405 0.125 × 0.405 ≈ 0.051
美味しい (文書A) 1 / 8 = 0.125 df(美味しい)=2 → ln(3/2) ≈ 0.405 0.125 × 0.405 ≈ 0.051
ご飯 (文書A) 2 / 8 = 0.250 df(ご飯)=2 → ln(3/2) ≈ 0.405 0.250 × 0.405 ≈ 0.101
散歩 (文書C) 1 / 7 ≈ 0.143 df(散歩)=1 → ln(3/1) ≈ 1.098 0.143 × 1.098 ≈ 0.157
考察：なぜ「散歩」の方が「ご飯」より重要度が高いのか？

「ご飯」は文書Aで2回も使われていますが、文書AとBの両方に出現するため、IDF（レア度）は 0.405 と低くなります。一方で「散歩」は文書Cで1回しか出現しませんが、他のどの文書にも出現しないため、IDFは 1.098 と約2.7倍高くなります。このレア度の差により、最終スコアは「散歩」の方が高くなり、その文書を象徴する言葉として評価されます。

コンピュータ処理の流れ

Step 1: 形態素解析

日本語文章を単語単位に分解。品詞と基本形（原形）を判定します。

Step 2: フィルタ＆揺れ補正

「です」「は」等の不要語を除去。シノニム置換やアルファベット大文字小文字の標準化を実施。

Step 3: 高度な統計計算

TF-IDFの計算、否定表現を考慮した感情反転スコアリング、Jaccard共起係数の計算を一括高速処理。

Step 4: 可視化と探索

ワードクラウド、インタラクティブな Plotly グラフや共起ネットワーク図に出力して探索します。
